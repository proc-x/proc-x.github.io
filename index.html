言葉の定義 Key Definitions

AIエージェントとマイクロエージェント AI Agents and Micro Agents

PROC.Xでは、AIエージェントを「判断しながら処理を進めるソフトウェア」、マイクロエージェントを「役割を一つに絞った小さなAIエージェント」として定義しています。巨大な仕組みを一気に作るのではなく、小さく作って検証しながらつなげていきます。 At PROC.X, an AI agent is software that makes decisions while moving work forward, and a micro agent is a small AI agent focused on a single responsibility. Rather than building one massive system, we build small, verifiable pieces and connect them over time.

🧠

AIエージェントAI Agent

入力内容や業務ルール、外部ツールの結果を見ながら、次に何をするかを判断して処理を進めるソフトウェアです。生成AIを使う場合もあれば、既存システムやルールベースの処理と組み合わせる場合もあります。 Software that decides what to do next based on inputs, business rules, and external tools. It may use generative AI, or combine it with existing systems and rule-based logic.

🧩

マイクロエージェントMicro Agent

AIエージェントの中でも、「見積書を作る」「メールを分類する」「台帳を更新する」のように、一つの役割だけを担当する小さな単位です。責任範囲が明確なので、検証・改善・置換がしやすくなります。 A smaller unit of an AI agent that handles one responsibility only, such as drafting a quote, classifying email, or updating a ledger. Clear scope makes it easier to verify, improve, and replace.

🎯

一つのことだけを担当するOne Responsibility Each

「見積書を作る」「メールを送る」「台帳を更新する」など、一つの処理だけに特化します。責任が明確なので、動作確認が容易で、問題が起きても影響範囲が限定されます。 Each agent focuses on just one thing — drafting a quote, sending an email, or updating a ledger. Clear responsibility makes testing easy and limits the blast radius if something goes wrong.

🔗

つなげることで大きくなるConnect to Scale

単独では小さなエージェントも、つなげることで複数ステップを自動化できます。最初は1つ、確認しながら少しずつ10個・20個と広げていきます。 A single agent is small, but connected agents can automate multi-step workflows. Start with one, expand to 10 or 20 as confidence grows.

🛡️

要所で人間が確認するHuman Review at Key Points

重要な判断が必要な場面では、エージェントは処理を止めて人間に確認します。これがHuman in the Loopです。業務事故を防ぎながら自動化を進められます。 At critical junctures, the agent pauses and requests human review. This is Human in the Loop — preventing errors while still advancing automation.

It also integrates more naturally with generative AI and agent-based AI, and may reduce licensing and maintenance costs.

社内データの選定・最適化Prepare Internal Knowledge for Agents

That includes preparing data for RAG and restructuring it into units that are easier to retrieve and use.

🌱

既存エージェントの育成Improve Existing Agents

すでに動いているエージェントに対しても、RAG設定、プロンプト調整、評価観点の整理、運用監視の見直しを行えます。「作ったが精度が安定しない」「現場で使いづらい」といった段階から改善できます。 We also improve agents that are already running by refining RAG settings, prompts, evaluation criteria, and operational monitoring. This helps when an agent exists but quality or usability is still unstable.

段階的な拡張 Gradual Expansion

一つから始めて、社内全体へ From One Process to Company-Wide

一度に全部を自動化しようとする必要はありません。まず一つの処理から始め、確認しながら徐々に広げていきます。成功体験を積み重ねながら、社内のエージェント化を拡張していくのが私たちのやり方です。 You don't need to automate everything at once. Start with one process, verify it works, then expand. We build on each success as we roll out agents across your organization.

Phase 1

まず一つをエージェント化Automate One Process

最も繰り返しが多い、または時間のかかる処理を一つ選んで始めます。動作を確認しながら、チームが慣れるまでゆっくり進めます。 Pick the most repetitive or time-consuming process and start there. Verify carefully and let the team get comfortable before moving forward.

最初のエージェントが安定したら、前後の処理も自動化してつなげます。SaaS APIやFaaSスクリプトで既存ツールと連携します。 Once the first agent is stable, automate surrounding steps and connect them. Integrate with existing tools via SaaS APIs and FaaS scripts.

  • SaaS API・FaaS連携の実装SaaS API & FaaS integration

業務事故を防ぐ仕組み Preventing Business Errors

Human in the Loop

完全自動化は目的ではありません。「エージェントに任せる処理」と「人間が判断すべき場面」を明確に分けることが、安全な自動化の核心です。 Full automation isn't the goal. Clearly separating what agents handle from what requires human judgment is the key to safe, reliable automation.

  • Gate A
    対外送付前レビューPre-Send Review

    顧客への見積・契約書など、外部へ送る前に担当者が確認しますStaff reviews quotes and contracts before anything is sent to customers

  • Gate B
    高リスク変更の承認High-Risk Change Approval

    契約条件・金額・法規制に関わる変更は責任者が承認しますContract terms, pricing, or regulatory changes require approver sign-off

  • Gate C
    例外時のエスカレーションException Escalation

    エージェントが対処できない例外は、人間にエスカレーションして判断を仰ぎますWhen agents encounter exceptions they can't handle, they escalate to humans for a decision

よくある質問FAQ

小さな会社でも始められますか？ Can small companies get started?

はい。むしろ小さな会社の方が、一つの処理を特定しやすく、効果を早く実感できます。まず「一番時間がかかっている処理は何か」をお聞かせいただくところから始めます。 Absolutely. Smaller companies can often identify specific pain points more easily and see results faster. We start by asking: "What one process takes the most time?"

既存のシステムを変える必要はありますか？ Do we need to change our existing systems?

基本的には不要です。現在使っているメール、SaaS、クラウドストレージをそのまま活かしながら、エージェントを追加していきます。必要に応じてAPIや連携設定を行いますが、システムの置き換えはしません。 Generally not. We add agents on top of the email, SaaS, and cloud storage you already use. We configure integrations as needed, but don't replace existing systems.

エージェントが間違えたらどうなりますか？ What happens if an agent makes a mistake?

重要な処理の前にはHuman in the Loopゲートを設けます。エージェントが実行する前に人間が確認できる仕組みがあるため、致命的な業務事故を防げます。また、すべての処理にログを残すため、問題が起きても追跡・修正が可能です。 We place Human in the Loop gates before critical operations, so humans can review before agents execute. All operations are logged, so issues can be traced and corrected.

どのくらいで効果が出ますか？ How soon will we see results?

最初のエージェントは、対象処理の選定から実装まで通常2〜4週間程度です。一つ動き始めれば、その処理にかかっていた時間が即座に削減されます。その後は横展開していきます。 The first agent typically takes 2–4 weeks from process selection to implementation. Once running, time savings are immediate. From there, the approach expands.

何から相談すればいいかわかりません I'm not sure what to discuss first

「この処理が面倒だ」「この作業に時間がかかりすぎる」というお話だけで十分です。技術的な知識は必要ありません。ヒアリングを通じて、最初の一歩を一緒に見つけます。 All you need is "this process is tedious" or "this task takes too long." No technical knowledge required. We'll find the right first step together through conversation.
