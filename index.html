AIエージェント・Python・GASを組み合わせる理由

AIエージェントは「判断」、Pythonは「データ処理と外部連携」、GASは「Google Workspace実行」を担当。 役割を分離すると、運用品質と変更スピードが同時に上がります。

  1. 入力統合: Gmail/Forms/CSVをPythonとGASで集約
  2. 判断分担: Planner・Worker・Reviewerのマルチエージェントで処理方針を決定
  3. 実行: PythonがAPI/DB連携、GASがSheets/Drive/Calendar更新を担当
  4. 監視: KPIを自動集計し、閾値割れ時に改善チケットを自動生成

マルチエージェント仕様

Planner Agent

Worker Agent

計画に沿ってPython/GASのジョブを呼び出し、処理を実行します。

Reviewer Agent

Ops Agent

実行ログとKPIを監視し、異常時に原因候補と改善タスクを起票します。

効果を見誤らないための指標

時間短縮だけで評価すると、品質低下や属人化を見逃します。4つの視点で評価するのが実務的です。

  • Speed: 1件あたり処理時間、リードタイム
  • Quality: 差戻し率、転記ミス率
  • Stability: 失敗率、再実行率、例外率
  • Adoption: 現場利用率、手動迂回率
KPIが閾値を下回った場合、マルチエージェントが原因候補を要約し、 PythonジョブやGASスクリプトの修正チケットを自動生成する運用まで設計します。
